# Nile v0.16.13

## Highlights

- Fixed packaged desktop sign-in flows so Nile can recover when local CLI installs are duplicated, broken, or hidden behind a different Finder `PATH`.
- Added a per-agent local CLI command path control in `Agent home` so you can see and override which executable Nile will use on this Mac.
- Improved desktop runtime diagnostics for local agent installations, including clearer handling for broken CLI wrappers and missing packaged binaries.

## Included Changes

- `Sign in with OpenAI` in packaged desktop builds now opens the browser more reliably and avoids falling back to stale or broken Codex CLI wrappers.
- Codex, Claude, Gemini, Cursor, and OpenClaw now expose their resolved local CLI command path in desktop settings, with a local override that stays on this machine only.
- Desktop command resolution now detects common `~/.nvm` installs and prefers working agent runtimes over unusable wrappers when possible.

## Upgrade Notes

- If Nile picks the wrong local executable on this Mac, open the agent detail page and set the `CLI command path` explicitly for that agent.
