# Nile v0.17.4

## Highlights

- Recovered desktop session refresh more reliably when OpenAI or Gemini sign-in tokens have expired.
- Allowed Nile to fall back from silent token refresh to a real re-login flow instead of getting stuck on repeated unauthorized quota reads.

## Included Changes

- OpenAI session quota failures that return unauthorized now trigger the shared current-session recovery path, so Codex-backed connections can prompt a fresh sign-in and then retry quota.
- Gemini session recovery now tries a silent CLI refresh first and switches to an interactive Gemini sign-in when Google requires full re-authentication.
- Desktop recovery flows now preserve browser-open behavior and agent runtime command overrides during background session repair, which keeps re-login consistent with the existing add/import flows.
