# Nile Desktop v0.15.1

## Highlights

- Packaged Nile desktop builds can now check public GitHub Releases for updates and install a newer version from inside the app.
- Settings now show the current desktop version, let you run an explicit update check, and surface an `Update` action when a newer version is ready.
- Desktop releases now ship with repository-owned release notes instead of generated GitHub notes.

## Included Changes

- Added packaged auto-update support for public `GitHub Releases` on desktop-supported platforms.
- Added a compact settings row for version visibility and manual update checks.
- Kept update failures non-blocking so the app still launches and stays usable if the release feed becomes unavailable.
- Added a required `release-notes/<tag>.md` source of truth for desktop releases and wired the release workflow to publish that content as the GitHub Release body.

## Upgrade Notes

- In-app updates only apply to packaged desktop builds and only after this release has been installed once.
- Future desktop releases must include a matching `release-notes/v<semver>.md` file before the release workflow will publish artifacts.
