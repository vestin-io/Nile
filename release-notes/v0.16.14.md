# Nile v0.16.14

## Highlights

- Unified desktop quota wording and polished the quota display so remaining limits read more clearly across connections and agent surfaces.
- Added a per-connection quota focus control, so you can pin which quota window Nile highlights instead of always following the tightest limit.
- Hardened Gemini quota recovery and diagnostics when saved or local Gemini CLI credentials drift or expire.

## Included Changes

- Desktop quota summaries, tray text, and related UI now consistently use `quota` wording instead of mixing `usage` and `quota`.
- Connection quota panels now let you locally choose which quota row should drive the headline summary for that saved connection.
- Gemini quota reads now retry more safely when the saved session is stale, and Nile logs more actionable detail when Gemini CLI refresh fails.

## Upgrade Notes

- Gemini quota recovery still depends on what the installed Gemini CLI can refresh silently on this Mac. If Gemini requires a full browser re-login, Nile now reports that state more clearly, but you may still need to re-authenticate in Gemini CLI manually.
