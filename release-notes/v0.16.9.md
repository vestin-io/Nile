# Nile v0.16.9

## Highlights

- Fix manual desktop update checks so Nile no longer reports "already up to date" when a newer GitHub release is actually available.
- Make desktop-installed updates quit the app fully before restart so the new build can replace the running bundle cleanly.
- Fix Claude desktop sign-in so Nile opens a real Terminal window for interactive login instead of trying to inherit a non-interactive Electron main-process terminal.

## Included Changes

- Manual update checks now compare the packaged app version with Electron's public GitHub update feed before deciding that no update is available.
- The desktop update install path now exits the tray-backed app explicitly before calling Electron's install-and-restart flow.
- Claude add-connection sign-in now runs in an isolated temporary Claude home, so adding a Claude session does not overwrite the account already active in the user's local Claude setup.
- Desktop add-connection screens now explain whether sign-in will continue in the browser or in Terminal and show a matching waiting state while the flow is in progress.
