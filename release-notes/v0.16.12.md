# Nile v0.16.12

## Highlights

- Reduced repeated macOS keychain prompts caused by Nile-managed API key shell exports.
- Simplified the desktop update prompt so it only appears when an update is ready to install or when an update check fails.
- Improved desktop localization across the menu bar, tray menu, and language selector.

## Included Changes

- Desktop-managed API key connections now avoid wiring shell profile exports unless an enabled agent actually needs an env-backed runtime path, which cuts down unnecessary keychain access during shell startup.
- Existing menu bar and tray surfaces now follow the selected desktop language more consistently, including tray context menu labels and status actions.
- The language selector now shows each language in its own native name, such as `English` and `中文`, instead of translating language names through the current UI locale.
- The floating update prompt no longer shows a fake downloading progress bar or extra status icon styling; it stays focused on plain text and action buttons.
