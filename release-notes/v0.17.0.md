# Nile v0.17.0

## Highlights

- Added Windows desktop release packaging and secure credential storage support alongside the existing macOS desktop app.
- Introduced encrypted migration packages so you can securely back up, move, and restore saved connections between machines.
- Simplified desktop credential storage into one machine-level mode, reducing setup drift and making unlock behavior more predictable.

## Included Changes

- GitHub desktop release builds now produce both signed macOS artifacts and Windows installer artifacts.
- Desktop can now export selected connections into encrypted `.nilevault` migration packages and import them back with preview, partial selection, and duplicate handling.
- System secure storage and encrypted local storage are now machine-level choices instead of per-connection toggles.
- Quick setup, connections, and agent surfaces now share the same credential-storage rules and import/export flows.

## Upgrade Notes

- If an existing local state contains a mix of secure-storage modes from older builds, Nile will require a local reset before continuing with the new single-mode model.
- Import/export currently ships on desktop surfaces first; CLI parity is planned separately.
