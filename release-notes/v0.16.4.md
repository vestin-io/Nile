# Nile Desktop v0.16.4

## Highlights

- Fixed the desktop release regression where CLI tests could fail during Cursor connection import on clean CI runners.
- Hardened Cursor usage auto-bind so CLI connection flows resolve local Cursor state from the active command context instead of ambient machine defaults.

## Included Changes

- Fixed the CLI Cursor usage follow-up path so importing a current Cursor session can auto-bind usage consistently in local runs and GitHub Actions.
- Removed an implicit dependency on a developer machine keychain item from the Cursor import regression test by making the Chrome safe-storage fixture explicit.
- Kept the broader desktop connection and plugin-registration improvements from v0.16.3 intact while repairing release reliability.

## Upgrade Notes

- This is a stability follow-up to v0.16.3. No manual migration is required for existing desktop connections.
