# Nile v0.17.2

## Highlights

- Fixed Cursor quota sync failures when a saved connection uses Nile's encrypted local storage instead of the system keychain.
- Improved the Agents page so connection switches show immediate in-place loading feedback instead of looking stuck while the backend apply flow finishes.
- Tightened desktop connection-switch state handling so loading only clears after the new current connection is actually reflected in app state.

## Included Changes

- Cursor quota bindings now follow the active connection credential backend instead of silently defaulting back to system secure storage.
- Damaged or unsupported persisted Cursor quota backend values now fail closed instead of reading from the wrong credential store.
- The Agents list current-connection selector now shows the pending target connection, swaps the chevron for a spinner during the switch, and keeps usage in a loading state until refreshed data arrives.
- Agent card switching feedback is lighter and more local to the selector, which makes connection changes feel responsive without blocking the rest of the page.
