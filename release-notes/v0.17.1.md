# Nile v0.17.1

## Highlights

- Added first-class OpenCode support, including OpenAI API key and OpenAI subscription session reuse.
- Improved agent setup flows so OpenCode can reuse compatible saved connections instead of forcing a new connection flow every time.
- Reduced desktop agent-switch and quota-refresh overhead, especially around repeated full-state refreshes.

## Included Changes

- Nile can now detect, import, apply, and roll back OpenCode selections through the desktop agent surface.
- OpenCode setup now keeps its global config in sync more reliably, including provider selection, model selection, and current connection reuse from existing OpenAI-compatible connections.
- Desktop model saves no longer trigger redundant refresh chains or unrelated quota reads.
- Desktop quota updates now use a lighter-weight automatic refresh path that reuses cached current-connection state instead of rebuilding all agent state on every interval.
- Desktop logs are now capped to 10 MB and old sibling log files are cleaned up automatically.

## Upgrade Notes

- This release continues to write OpenCode changes to the global OpenCode configuration. Project-level `opencode.json` overrides are not managed by Nile yet.
