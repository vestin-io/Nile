# Nile v0.17.6

## Highlights

- Fixed desktop session reauthentication so expired Codex, Claude, and Gemini connections recover more predictably without kicking off unnecessary sign-in prompts during routine usage refresh.
- Simplified the quick setup guide by removing the inline `Import package` action from the unsaved-local-setup card, keeping that surface focused on saving detected local setups.

## Included Changes

- OpenAI session-backed connections now try to sync the current local Codex session before asking you to sign in again, which reduces false reauthentication prompts when the saved connection token has drifted but the local session is still valid.
- Manual and background quota refresh no longer open interactive login flows on their own; unauthorized session-backed connections stay in a clear reauthentication-required state until you explicitly choose to fix them.
- Saved Gemini connections now use a dedicated refresh path for the current local Gemini session instead of routing reauthentication through the generic add-account login flow.
- Quick setup no longer mixes portability import with the “save your detected local setups” guidance card, making the onboarding message easier to scan.
