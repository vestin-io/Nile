# Nile v0.17.5

## Highlights

- Fixed OpenAI add-connection defaults so Codex-backed session imports now start from the expected current-session path instead of falling back to browser login.
- Added a clearer desktop reauthentication flow for expired OpenAI, Claude, and Gemini session connections without letting background quota refresh trigger browser sign-in on its own.

## Included Changes

- Desktop quota refresh now keeps unauthorized session failures visible and labels them as reauthentication-required, so saved connections show an actionable state instead of silently collapsing to unknown usage.
- Connection detail now surfaces a dedicated `Reauthenticate` action for supported session-backed connections and routes it through the same encrypted-local unlock recovery used by other connection mutations.
- Connection list and detail usage summaries were tightened so generic unavailable quota states stay localized, while detailed provider-specific guidance remains available in the connection detail alert and quota panel.
