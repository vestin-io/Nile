# Nile Desktop v0.16.1

## Highlights

- Fixed OpenClaw local OpenAI session imports so `Save to Nile` works again after a reset or after removing the old saved connection.
- Stopped OpenClaw-only OpenAI sessions from leaking back into Codex as if they were full shared Codex sign-in connections.
- Tightened desktop and core connection handling so broken or partial OpenClaw session state is treated more explicitly.

## Included Changes

- OpenClaw local `openai-codex` profiles now save as an OpenClaw-specific OpenAI session connection instead of pretending to be a full Codex-style OpenAI session.
- Re-imported OpenClaw local setups no longer require an `idToken`, which fixes the `Save to Nile` failure that appeared after clearing the old saved connection state.
- OpenClaw-only OpenAI session connections stay available to OpenClaw while staying out of Codex connection switching and shared-session assumptions.
- Connection matching, label generation, usage/model-catalog access, and desktop display text were updated so this OpenClaw-only session type behaves consistently across the app.

## Upgrade Notes

- If you previously reset local Nile state and still have an OpenClaw local setup, you can now save that setup back into Nile without recreating the session through Codex first.
