# Nile Desktop v0.15.5

## Highlights

- Stabilized connection switching across Codex, OpenClaw, and shared saved connections so local session changes no longer leave agents in confusing mismatched states.
- Fixed a set of desktop connection editing issues, including empty provider summary cards, session sign-in hangs, and browser-safe build regressions in dev mode.
- Hardened managed environment handling for OpenClaw-backed API key flows, including reset behavior, shell export sync, and safer startup repair.

## Included Changes

- OpenClaw now keeps using its own saved connection state after apply instead of continuously depending on the current Codex live auth session.
- Known local setups now auto-sync the saved selection for the same agent when Nile can uniquely match the live setup to an existing saved connection.
- Editing or adding custom/gateway-style connections no longer shows an empty provider summary shell when there is no provider metadata to display.
- OpenAI/Codex session sign-in now runs asynchronously and prefers the current process PATH, avoiding frozen desktop windows and broken login attempts caused by stale CLI install paths.
- Managed `NILE_*` environment syncing now avoids redundant Keychain rewrites, batches reset cleanup, and applies safer shell profile updates.

## Upgrade Notes

- If you previously saw OpenClaw warnings about missing `NILE_*` variables or Codex/OpenClaw session mismatches, restart the desktop app after upgrading so it can repair the managed environment and saved agent state.
