# Nile Desktop v0.16.0

## Highlights

- Improved desktop update UX so packaged builds clearly show when Nile is checking, downloading, ready to install, or unable to reach the update service.
- Added a global in-app update prompt so users no longer need to reopen Settings to understand what is happening with a background update.
- Smoothed several desktop connection and agent flows that previously led to confusing or fragile local state.

## Included Changes

- Packaged desktop builds now show clearer update states in Settings, including explicit background download, install-ready, and retryable error states.
- Nile now shows a global update card for:
  - update downloading
  - update ready to install
  - update check failures
- The update prompt now offers direct actions to:
  - view release changes
  - restart and install when the update is ready
  - retry an update check after failure
- OpenClaw no longer keeps depending on the current Codex live auth session after apply; it stays bound to its own saved Nile connection.
- Known local setups now auto-sync the saved selection for the same agent when Nile can uniquely match the current live setup to an existing saved connection.
- Managed environment syncing now avoids unnecessary Keychain rewrites and batches reset cleanup more safely.

## Upgrade Notes

- To test the new updater experience, run a packaged Nile build older than `0.16.0`, then open Settings or let the app check for updates in the background.
- Download progress is currently shown as a background download state rather than a numeric percentage, because the current updater stack does not expose a reliable percentage event.
