# Nile v0.17.3

## Highlights

- Reduced steady-state desktop usage polling so idle sessions refresh usage every 5 minutes instead of every minute.
- Kept startup refresh behavior intact, so the app still loads fresh usage data when a desktop session begins.

## Included Changes

- The desktop usage cache TTL now also drives a slower automatic background refresh cadence, which cuts unnecessary quota polling during long-running idle sessions.
- Manual and startup-triggered refresh paths continue to work as before, so usage data can still be refreshed immediately when needed.
