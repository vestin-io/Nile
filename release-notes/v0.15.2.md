# Nile Desktop v0.15.2

## Highlights

- Fixed saved-connection sign-in failures on Intel Macs caused by the packaged keychain helper shipping with an incompatible architecture/runtime combination.
- Improved compatibility for macOS 12 desktop installs by lowering the bundled keychain helper deployment target to match supported Monterey systems.
- Preserved clearer credential-sync diagnostics so failed local keychain writes are easier to trace during support and local testing.

## Included Changes

- Rebuilt the bundled keychain helper as a universal macOS binary so both Apple Silicon and Intel desktop builds can launch the same helper safely.
- Lowered the helper runtime target to macOS 12.0, which fixes dynamic-linker failures seen on Monterey machines while keeping the desktop app signed and notarized for both architectures.
- Improved local credential-sync error reporting so helper startup failures surface the underlying system error instead of collapsing to a generic keychain command failure.

## Upgrade Notes

- Intel and Monterey users should replace older installed Nile apps with the new v0.15.2 package before re-testing saved OpenAI/Codex session connections.
