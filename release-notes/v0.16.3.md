# Nile Desktop v0.16.3

## Highlights

- Reworked desktop connection setup so agent-specific add flows only show the providers that actually belong to that agent.
- Added a first-class Gemini sign-in flow for creating a new Gemini CLI connection instead of only reusing an existing local session.
- Improved plugin registration boundaries and runtime wiring to make desktop connection and session flows more reliable.

## Included Changes

- Fixed agent detail add-connection pages so Codex, Cursor, Gemini, and other agents open into the right scoped provider list instead of the global provider catalog.
- Added dedicated Cursor and Gemini provider handling in the desktop add-connection experience, including better connection-method defaults.
- Hardened Gemini desktop sign-in on macOS by launching Terminal correctly, isolating login state for new-account sign-in, and completing the save flow after OAuth succeeds.
- Moved more runtime registration and usage behavior into builtins/registry-driven wiring, reducing drift across desktop and CLI surfaces.
- Sanitized test fixtures and related build notes so personal email addresses are no longer stored in the repository.

## Upgrade Notes

- Existing connections keep working as before. This release mainly improves how new desktop connections are created and scoped from agent-specific entry points.
